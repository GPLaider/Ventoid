# Code of Conduct

Ventoid is a small free software project. The goal is to keep discussion useful, technical, and safe for people reporting bugs, testing devices, and contributing code.

## Expected Behavior

- Be respectful and direct.
- Assume good faith when someone reports a problem.
- Keep criticism focused on code, documentation, builds, devices, or process.
- Provide technical details when disagreeing.
- Respect user privacy, especially logs, screenshots, device identifiers, and storage contents.

## Unacceptable Behavior

- Harassment, insults, threats, or personal attacks
- Publishing private information without permission
- Spam, off-topic promotion, or repeated low-effort noise
- Pushing users toward unsafe USB write steps without clear warnings
- Pressuring the project to add closed-source, adware, telemetry, or unverifiable binary components

## Enforcement

Maintainers may edit, hide, or delete comments and issues that violate this policy. Maintainers may also block users who repeatedly ignore these rules.

For ordinary bugs and feature requests, use GitHub issues. For sensitive security reports, follow `SECURITY.md`.

## Scope

This policy applies to Ventoid project spaces, including GitHub issues, pull requests, release discussions, and related project support threads.
